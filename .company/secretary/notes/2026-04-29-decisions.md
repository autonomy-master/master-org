# 意思決定ログ - 2026-04-29

## 決定事項

### 自律型マルチAI専門家組織 v1 のスコープ確定

- **背景**: オーナーの北極星ビジョン「各分野の専門家AIを部屋ごとに配置し、半永続的に会議・実行・報告させる」を実現するため、v1のスコープを協議。
- **判断**: v1は **`money/`（収益化部）と `writing/`（執筆部）の2部屋** を高品質に立ち上げる。動作・品質を確認後、`engineering/` → `anime/` / `music/` / `art/` を順次追加。
- **理由**:
  - 6部屋同時建造は「動くけど浅い」状態に陥りやすい
  - ディレクトリ名「MAKE-MONEY」と整合する `money/` を最優先
  - `writing/` は記事・本のニーズに直結し、money 部屋のコピーや LP 草案とも相互送客できる
- **対応部署**: money, writing（新設）
- **フォローアップ**:
  - [ ] money/ 部屋構築（CLAUDE.md + 5専門家 + meeting protocol + outputs/）
  - [ ] writing/ 部屋構築（CLAUDE.md + 5専門家 + meeting protocol + outputs/）
  - [ ] master `.company/CLAUDE.md` に部署追加
  - [ ] 各部屋で初回会議を smoke test として実行
  - [ ] cron スケジュール設定（朝9時 / 夕18時 / 深夜3時 など）

### 外部生成APIの予算方針

- **背景**: アニメ・音楽・芸術部屋は外部生成API（Suno, ElevenLabs, 画像AI, Runway等）なしでは「脳だけ動いて手足が動かない」状態になる。
- **判断**: **「全部いいよ」** = 画像・音楽・動画の生成APIをフル解禁可（月額数万円コスト想定）。ただし v1 スコープには含めず、money + writing が稼働してから順次解禁。
- **理由**:
  - 収益が見え始めてから生成APIに投資する方が ROI を測定しやすい
  - v1 で組織運営パターンを固めてから生成APIを足す方が事故が少ない
- **対応部署**: 秘書（予算解禁タイミングの提案役）
- **フォローアップ**:
  - [ ] money 部屋から最初の収益または検証結果が出たタイミングで、anime/music/art 部屋構築 + API連携の提案を秘書から行う

### 自律性の境界線（暫定）

- **背景**: 「寝てる間も勝手に動く」「自律判断OK」とのオーナー意向。
- **判断**: 各部屋は cron で定期発火し自律的に会議・成果物生成・記録を行う。ただし以下は **秘書が止めて朝レビュー** に回す:
  - 外部発信（SNS投稿の公開、ブログ公開、メール送信）
