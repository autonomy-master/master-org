## 公開前の必須チェック（毎回）

### 1. ファイル本文確認
- meta ブロックの「※ 本記事は連載第N回」表記が正しいか
- 連載リンク末尾の URL が最新か（前回までの公開済み記事すべてリンクされているか）
- AI 透明性表記が末尾に存在するか

### 2. note 編集画面での貼り付け確認
- 太字（**...**）が note の太字記法と整合するか（note は \`**\` で太字 OK）
- 引用ブロック（`>`）が note 表示で意図通りか
- 表（| ... |）が note 表示で崩れていないか

### 3. タイトル設定
- note のタイトル欄には **第N回 タイトル** ではなく **タイトルのみ** 入力（連載性は本文先頭で示す）

### 4. タグ設定
- 共通: `#AI` `#個人開発` `#SaaS` `#AIエージェント` `#連載`
- 第5回追加: `#scope` または `#組織論` （任意）

### 5. 公開後すぐ
- URL を私（このチャット）に共有
- → 自動で:
  - Tweet N 発射
  - README 更新
  - 連載 INDEX 更新
  - GitHub commit + push
  - Discord に live 通知

## 巨匠 PM の推奨

**モード C（Day 2 連投 + Day 3 単発）**:
- Day 2: 第3-4回連投（連続物語の重み）
- Day 3: 第5回（AI 自己批判というメタ的な独立記事）

理由:
- 第3-4回は内容が連続（Live モード切替 → 数字0で走る）、連投で読み通しやすい
- 第5回は別主題（AI 自己批判）、距離を取って単独で読まれる方が刺さる
- 3日で連投5本（既公開2 + 連投3）= 強いリズム
