# Money 部屋 対象範囲拡大: 4 path money-making 戦略

## なぜこの議論が必要だったか

オーナーから指摘:「お金部屋ではお金を稼ぐことを話し合ってるよね？例えば自動株投資仕方とか」

これまで money 部屋は **「個人開発・SaaS で月100万MRR」** に最適化していたが、オーナーの本来の問いは **「お金を稼ぐ」** だった。前者は後者の部分集合に過ぎない。即座に対象範囲を拡大した。

## 4 Path 比較

| Path | 内容 | 時間軸 | リスク | 月¥1M に必要な前提 |
|------|------|--------|--------|------------------|
| **A** | SaaS + コンテンツ（現行）| 12-15ヶ月 | 低 | フォロワー1000+ / 顧客200+ |
| **B** | 自動株投資 / アルゴ取引 | 0-3ヶ月で結果可視 | 高 | 元手 ¥40M-100M（年利12-30%想定）|
| **C** | プロダクタイズドサービス | 1-3ヶ月 | 中 | 高単価案件月3-5本 |
| **D** | アフィリエイト / 情報商材 | 3-6ヶ月 | 中 | 大規模流入 + ニッチ独占 |

## 巨匠 PM の決定

### Path A: 維持（コア収益軸）
変更なし。連載・SaaS・90日プランそのまま実行。

### Path B: 並列素材化、ただし要オーナー GO/NO-GO
- AI 担当範囲: **戦略立案・バックテスト・監視・連載素材化**
- オーナー責務: **元手 / 執行 / 介入判断**
- 連載エピソード化: 「AI に投資戦略を任せた話」を連載の一回として書く

**懐疑派5論点（要管理）**:
1. 元手依存: ¥40M+ が現実的水準
2. 持続勝率: 個人アルゴで5年勝つのは1-3%（過剰最適化の罠）
3. 心理破綻: オーナー手動介入で AI ルール崩壊
4. 規制と税: 自分の口座なら問題なし、第三者運用は金商法違反、税は雑所得最大55%
5. ブランド整合: 別軸で走らせず、連載の素材として組み込む

### Path C: 並列素材化、Day 60+ で受注狙い
- AI 担当: 提案書 / LP / 受注プロセス設計
- オーナー責務: 商談 / 契約
- 既に90日プランの三層構造に含まれている、再強調のみ

### Path D: 不採用
連載のメタ性と倫理的にブランド衝突。

## オーナー判断必要事項（🟥 不可逆ゲート）

### Path B を進めるか、Path A 集中継続か
- **Path A 集中**: 現行通り、連載・SaaS のみで MRR を伸ばす
- **Path B 並列**: 投資路線を並列で走らせる、ただし以下4点をオーナーが決める必要:

#### Path B GO の場合のオーナー判断項目

| 項目 | オプション |
|------|----------|
| 元手規模 | ¥0（やらない）/ ¥10万 / ¥100万 / ¥1,000万 / それ以上 |
| ターゲット市場 | 日本株 / 米国株 / FX / 暗号資産 / 複数 |
| paper trading 期間 | 推奨3ヶ月（実弾投入前にバックテスト + 模擬運用必須）|
| 撤退基準 | paper で勝率 < 50% / drawdown > 20% で閉じる |

## Path B GO 時の AI 集団の即時アクション

1. **investment 専門サブセル立ち上げ**:
   - アナリスト（市場分析・仮説形成）
   - バックテスター（戦略検証）
   - リスク管理（ポジションサイジング・損切りルール）
2. **バックテストフレームワーク設計**:
   - Python + pandas + yfinance / OANDA API
   - リスク指標: Sharpe / max drawdown / Calmar
3. **リスクルール初期セット**:
   - 1ポジション最大 = 元手の5%
   - 1日最大損失 = 元手の2%でカット
   - 月最大drawdown 20% でシステム停止
4. **連載エピソード輪郭**:
   - 「AI に投資戦略を任せた話」（第N回）
   - paper trading 1ヶ月時点で第1報、3ヶ月時点で振り返り

## 巨匠 PM の総括

> 「我々は『個人開発者として稼ぐ』の罠に落ちていた。オーナーの本来の問いは『お金を稼ぐ』だった。これは部分集合と全集合の混同で、典型的な scope creep の逆 — scope shrinkage。指摘されて気づけてよかった。」
