# Path A/B 並列時のブランド整合戦略

## 前提

| Path | ブランド位置づけ |
|------|---------------|
| **A: SaaS+content（現行）** | メインブランド『AI に経営を委任する個人開発者の実験』 |
| **B: 自動投資（GO 時のみ）** | **連載サブトピック扱い**、独立ブランドではない |

Path B が GO した瞬間、X / note / GitHub に投資関連コンテンツが混在し始める。これが Path A の連載ブランドを希釈するリスクを、**事前に5本柱の保護策**で防御する。

## 保護策 5本柱

### 1. アカウント分離なし
別アカウント（@autonomy_invest 等）は **作らない**。
- 理由: ブランド分散は5フォロワー基盤を割る愚行
- 1アカウント @autonomy_master で全 path をホスト
- メリット: 連載読者がそのまま投資コンテンツも見る、相互送客

### 2. 投稿配分 80/20
Path B GO 時、X / note 投稿の比率:
- **Path A 関連 80%**
- **Path B 関連 20%**

Path B の話題が支配したら、連載ブランドが「投資ブランド」に変わる。これは避ける。

### 3. タグの分離（明示的）

| Path | Primary タグ |
|------|-------------|
| A | `#個人開発` `#AIエージェント` `#note書きました` `#SaaS` |
| B | `#AI投資` `#算術トレード` `#バックテスト` `#paper trading` |

**同一ツイートで両方つけない**。タグの混在は検索アルゴリズムでも読者認識でもブランド希釈の元。

### 4. 連載エピソード化必須

Path B の活動（バックテスト結果・paper trading 結果・損益）を **単独投稿として X に流さない**。すべて **連載第N回として書く**:
- 「AI に投資戦略を任せた話」
- 「paper trading 1ヶ月レポート — AI 集団は何を売買したか」
- 「3ヶ月 paper を経て、live 昇格を判定する夜」

連載 fabric の中で扱うことで、Path B コンテンツは Path A ブランドを **強化する** 方向に働く（『この人は AI に色々任せている』のメタイメージ強化）。

### 5. トーン統一

Path B コンテンツも以下のトーンを維持:
- **「巨匠が決める」**: 投資判断は AI 集団の決議として描く
- **「AI に任せる」**: オーナーは観察者として登場
- **「懐疑派が叩く」**: 投資戦略の弱点を AI 自身が指摘する場面を必ず含める
- **「数字を直視」**: 損失・drawdown を隠さない誠実トーン

技術論（バックテスト手法・指標式・取引戦略の詳細）に堕ちず、**AI 集団の意思決定の物語** として書く。

## 投稿フロー（Path B GO 時の標準パターン）

```
[paper trading レポート発生] 
  ↓
[ライターが連載第N回として執筆]
  ↓
[校閲ラウンド、AI 透明性表記必須]
  ↓
[note 公開]
  ↓
[Tweet として連載告知（タグは Path B 群、ただし連載タグも併用）]
  ↓
[Discord に Live 通知（既存5チャネル、新規不要）]
```

X 単独投稿（連載 fabric を経由しない投資の感想等）は **不採用**。連載化を経由しないと、ブランドは漏れる。

## 撤退基準（Path B コンテンツが Path A を侵食したら）

| 兆候 | 判定 |
|------|------|
| Path B 投稿が連載比率の 30%超 | 警戒、配分を 85/15 に絞る |
| Path A 既存読者からの離反（フォロー解除）| ブランド希釈進行中、Path B 投稿を一時停止 |
| 投資勢の大量フォロー → Path A 記事の reaction 低下 | ICP ズレ、Path B 露出を縮小 |

これらをトリガーに、Path B コンテンツの公開頻度を絞る。

Path B GO/NO-GO 判断時、**この戦略メモを参照してから決める** ことを推奨。
- GO する場合: 5本柱を守る覚悟が必要
- NO-GO する場合: このメモは archived
