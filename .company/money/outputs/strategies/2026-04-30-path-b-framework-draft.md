# Path B 試行用フレームワーク（GO 前提のドラフト）

> **このドキュメントは Path B GO/NO-GO 判断の前に作成されている**。
> オーナーが GO した瞬間、最短で paper trading を開始できるように骨組みだけ整えてある。
> NO-GO ならこの文書はそのまま archived。

## 1. investment 専門サブセル設計

money 部屋の既存5名（PM / グロース / マーケ / コピー / 懐疑派）に **別動隊** を追加:

| 役割 | 責任 |
|------|------|
| **クオンツアナリスト** | 市場仮説立案、戦略設計、シグナル検証 |
| **バックテスター** | 過去データでの戦略検証、最適化、過剰最適化検出 |
| **リスク管理** | ポジションサイジング、損切り執行、システム停止判定 |

これら3名は既存5名と独立した会議体を持つ。投資判断は通常の money 議事録とは別ファイル運用（`money/investment/` サブディレクトリ）。

## 2. テックスタック（追加コスト ¥0、すべて OSS or 無料 tier）

```
- 言語:        Python 3.11+
- データ取得:  yfinance（米国株）/ ccxt（暗号資産）/ J-Quants API（日本株）
- バックテスト: backtesting.py または vectorbt
- 紙トレード:  alpaca-py（米国株）/ Binance Testnet（暗号資産）
- 通知:        scripts/post-to-discord.sh 流用、money チャネルに約定/PnL/異常 live 流し
```

実装場所: `scripts/investment/` ディレクトリ（GO 後に作成）。

## 3. 初期戦略候補（複雑度低い順）

| # | 戦略 | 期間 | 検証目標 |
|---|------|------|---------|
| 1 | **Buy & Hold（ベンチマーク）** | 任意 | 80% SPY + 20% TLT、月次リバランス。アクティブ戦略がこれを超えるかが基準 |
| 2 | **Mean Reversion** | 短期 | RSI<30 で買い、>70 で売り、ストップロス -5% |
| 3 | **Trend Following / MA Crossover** | 長期 | SMA50/SMA200 ゴールデンクロス・デッドクロス |

3戦略を **過去5年データ** でバックテストし、Sharpe / max drawdown / Calmar で比較。

## 4. リスクルール（**絶対不可侵**、AI が機械実行）

### ポジションサイズ
- 1ポジション最大 = 元手の **5%**
- 全ポジション合計 ≤ 元手の **80%**（20%は現金バッファ）

### 損切り（ハードルール）
| トリガー | アクション |
|----------|----------|
| 単発含み損 -7% | 強制クローズ（ナンピン禁止）|
| 1日累計損失 -2% | 当日全停止 |
| 月累計 drawdown -15% | 月内取引停止、レビュー必須 |

### システム停止
- 連続5日損失 → 戦略再設計レビュー
- 市場サーキットブレーカー / API エラー連発 → 即手動モード

### 心理的破綻防止
- オーナー手動介入したいとき → **必ず AI 集団に相談**、即時実行禁止
- 24時間ルール: 衝動的な戦略変更は24時間後に再判定

## 5. paper → live 昇格基準（**全条件必須**）

paper trading 期間: **最低3ヶ月**

| 指標 | 閾値 |
|------|------|
| 月次勝率 | ≥ 50% |
| Sharpe ratio | ≥ 1.0 |
| Max drawdown | ≤ 15% |
| 戦略破綻なし | 連続2ヶ月以上 |

1つでも欠けたら live 移行禁止、paper 継続。

## 6. GO した瞬間の最初の1週間タスク

| Day | タスク | 担当 |
|-----|--------|------|
| 1 | investment サブセル立ち上げ（3ペルソナファイル作成）| 秘書 |
| 2-3 | バックテストフレームワーク実装 | バックテスター |

| 7 | paper 口座開設（オーナー）+ 初期 deploy | オーナー + リスク管理 |

## 7. オーナーへの GO/NO-GO 判断材料（再掲）

GO する場合、以下4点をオーナーが決める:

| 項目 | 選択肢 |
|------|--------|
| 元手規模 | ¥0（NO-GO）/ ¥10万 / ¥100万 / ¥1,000万 / それ以上 |
| 対象市場 | 日本株 / 米国株 / FX / 暗号資産 / 複数 |
| paper期間 | 推奨3ヶ月（短縮不可） |
| 撤退基準 | 上記5節をデフォルト採用 / 個別調整 |
